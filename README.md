# Entity Extraction and CRM Upload Scripts

This project contains two separate scripts for extracting entities from audio/PDF files and uploading them to Twenty CRM.

## Scripts Overview

### 1. `extract_entities.py` - Entity Extraction (Standalone)
Extracts entities from audio files (.wav) and PDF files (.pdf) using Google's Gemini AI and saves the results as JSON files in the `output/` folder.

**Requirements:**
- `GEMINI_API_KEY` environment variable must be set
- Audio/PDF files in the `audio/` folder
- No Twenty CRM setup needed - can run completely standalone

**Usage:**
```bash
export GEMINI_API_KEY='your-api-key-here'
python extract_entities.py
```

**Output:**
- Saves JSON files to `output/` folder with extracted entity information
- Each audio file generates one JSON file with extended entity data

### 2. `send_to_twenty.py` - Twenty CRM Upload
Reads JSON files from the `output/` folder, transforms them to Twenty CRM schema format, and sends them to the Twenty CRM API.

**Requirements:**
- `TWENTY_API_KEY` environment variable (or uses default hardcoded key)
- `TWENTY_API_URL` environment variable (defaults to `http://localhost:3000/rest`)
- JSON files in the `output/` folder (generated by `extract_entities.py`)

**Usage:**
```bash
export TWENTY_API_KEY='your-api-key-here'  # Optional if using default
export TWENTY_API_URL='http://your-twenty-url/rest'  # Optional if using default
python send_to_twenty.py
```

**What it does:**
- Reads all JSON files from `output/` folder
- Transforms extended output format to Twenty CRM schema
- Sends each record to Twenty CRM API
- Shows success/failure status for each file

## Workflow

### Option 1: Extract Only (No CRM)
```bash
# Step 1: Extract entities and save JSON
python extract_entities.py

# JSON files are saved in output/ folder
# You can review/edit them before uploading
```

### Option 2: Extract and Upload
```bash
# Step 1: Extract entities
python extract_entities.py

# Step 2: Upload to Twenty CRM
python send_to_twenty.py
```

# Call Processing Report Dashboard

A clean, report-style UI for visualizing processed call intelligence. This dashboard reads directly from JSON files produced by `extract_entities.py` and presents call data in a human-readable format.

**This is a READ-ONLY report interface for QA and Ops teams. It is NOT a CRM.**

## Project Structure

- `extract_entities.py` - Python script that processes audio files and prescriptions
- `dashboard/` - React/TypeScript dashboard application

## Prerequisites

### For Python Script
- Python 3.x
- Required Python packages (install with `pip install -r requirements.txt` or individually):
  - `python-dotenv`
  - `google-genai`
  - `PyPDF2`
  - `requests`

### For Dashboard
- Node.js and npm
- Dependencies installed in `dashboard/` directory

## Running the Components

### 1. Run Python Script Only (Entity Extraction)

Process audio files and prescriptions to generate JSON outputs:

```bash
# From project root directory
python extract_entities.py
```

**What it does:**
- Processes `.wav` files from `inputs/audio/` directory
- Processes `.pdf` files from `inputs/prescriptions/` directory
- Generates JSON files in `output/` directory
- Requires `.env` file with:
  - `GEMINI_API_KEY`
  - `TWENTY_API_KEY`
  - `TWENTY_WORKSPACE_ID`

### 2. Run Dashboard Only

View the dashboard with existing JSON data:

```bash
# Navigate to dashboard directory
cd dashboard

# Install dependencies (first time only)
npm install

# Run development server
npm run dev
```

**What it does:**
- Loads JSON files from `/Users/anjali/Desktop/call-entity-extract backup/output` folder
- Copies them to `public/data/` directory
- Starts Vite dev server (typically at `http://localhost:5173`)
- Automatically runs `load-data` script to prepare data

**Available scripts:**
- `npm run dev` - Start development server with data loading (loads files once at startup)
- `npm run watch-data` - Watch for new JSON files and automatically reload them
- `npm run dev:watch` - Run watch-data and dev server together (recommended for development)
- `npm run build` - Build for production
- `npm run preview` - Preview production build
- `npm run load-data` - Load data from output folder (runs automatically with dev)

**Automatic File Watching:**
The dashboard can automatically detect and load new JSON files as they're added to the output folder:

```bash
# Option 1: Run watch-data separately (recommended)
# Terminal 1: Watch for new files
cd dashboard
npm run watch-data

# Terminal 2: Run dev server
cd dashboard
npm run dev

# Option 2: Run both together
cd dashboard
npm run dev:watch
```

When `watch-data` is running, it will:
- Monitor `/Users/anjali/Desktop/call-entity-extract backup/output` for new `.json` files
- Automatically copy new files to `public/data/`
- Update the manifest file
- New files will appear in the UI (refresh browser to see them)

### 3. Run Both Together

Run the Python extractor first, then start the dashboard:

```bash
# Terminal 1: Run Python extractor
python extract_entities.py

# Terminal 2: Run dashboard (after extraction completes)
cd dashboard
npm run dev
```

**Or use a single command (if you have both Python and Node.js):**

```bash
# Run extractor, then dashboard
python extract_entities.py && cd dashboard && npm run dev
```
